action 自动化部署

github 生成Personal access tokens

https://www.jianshu.com/p/d0d0e29caed1

部署

  • 创建一个私有仓库，用来存放源码，比如hugo-blog。

  • 创建一个名称为：用户名.github.io的仓库，用来存放hugo生成的静态网页代码

  • 生成personal access token（Action 需要有操作你的仓库的权限）

    点击头像 -> Settings -> Developer settings -> Personal access tokens -> Generate new token，

    只需要授予repo权限即可，记住这串TOKEN，网页关闭后就无法查看了。

  • 依次点击Setting->Secrets->Actions->New repository secrets，Name填ACCESS_TOKEN ,Value填刚刚获取的TOKEN

  • 上传博客源码到源码存放仓库

  • 对源码仓配置 github actions

    官方配置指导文档：https://gohugo.io/hosting-and-deployment/hosting-on-github/

    • 在根目录下创建配置文件 .github/workflows/gh-pages.yml，并配置action内容（/表示一层目录）

      name: github pages # 名字自取

on:
 push:
   branches:
     - main  # 这里的意思是当 main分支发生push的时候，运行下面的jobs，这里先改为github-actions

jobs:
 deploy: # 任务名自取
   runs-on: ubuntu-20.04	# 在什么环境运行任务
   steps:
     - uses: actions/checkout@v2	# 引用actions/checkout这个action，与所在的github仓库同名
     - name: Setup Hugo	# 步骤名自取
       uses: peaceiris/actions-hugo@v2	# hugo官方提供的action，用于在任务环境中获取hugo
       with:
         hugo-version: 'latest'	# 获取最新版本的hugo
         # extended: true

     - name: Build
       run: hugo --minify	# 使用hugo构建静态网页

     - name: Deploy
       uses: peaceiris/actions-gh-pages@v3	# 一个自动发布github pages的action
       with:
         external_repository: 用户名/用户名.github.io	# 发布到哪个repo
         personal_token: 填写上面步骤中创建的personal access token # 发布到其他repo需要提供上面生成的personal access token
         publish_dir: ./public	# 注意这里指的是要发布哪个文件夹的内容，而不是指发布到目的仓库的什么位置，因为hugo默认生成静态网页到public文件夹，所以这里发布public文件夹里的内容
         publish_branch: main	# 发布到哪个branch

      如果有 submodule,上面的需要加上几行，如下

      - name: Check out repository code
  uses: actions/checkout@v2
  with:

    fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod
- name: Setup Hugo

    • 配置完成后将配置文件上传到源码仓

  • 点击上侧一排的【actions】按钮，点击左侧的【deploy】，点右侧的【Run workflow ▽】切换到main分支，选择绿色的【Run workflow】 按钮 ，此时将成功部署。

FQA

Q：Error: Action failed with “The process ‘/usr/bin/git’ failed with exit code 128”

A：https://blog.saky.site/post/github-action-128/

  • 将源码仓库设置为私有

  • 重新生成personal access token

  • 回到通过生成的personal access token生成Secrets，Name填ACCESS_TOKEN ,Value填刚刚获取的TOKEN

  • 更新yaml中personal_token的值

自定义域名

为什么要自定义域名

腾讯云托管平台给每个用户提供一个免费的默认域名，比如

1

https://hugo-2g64t7rc4f3f7b5b-1313255125.tcloudbaseapp.com/

或者使用 github 托管代码给我们提供一个域名 https://username.github.io/

但这些默认域名要么太难记要么太low了，我们需要一个特殊的域名，比如www.chenyd.com，因此需要自定义域名。
